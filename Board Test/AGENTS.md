# AI 对话前置约束文档

## 核心要求

1. **禁止臆想硬件信息**：对于任何不清楚、不确定的硬件连接、引脚分配、外设参数，必须返回询问用户，不允许自行假设或猜测。

2. **对话结束后需要写入log日志**: 将ai操作记录在AI_log.md文件中，没有文档就创建，要求记录操作时间与操作的ai模型。 

3. **对2的补充**：当ai执行操作的时，请在对话完毕后询问用户是否完成功能，只有判断成功完成功能才记录。
 
4. 不要自动使用make检查编译错误，这会降低回复速度。
5. **每次执行对话前请阅读当前文档**

## 代码架构

1. BSP文件夹，存放底层BSP硬件驱动。inc存放.h文件，src存放.c文件。没有文件夹就自行创建

2. APP文件夹，存放应用层实现函数，包括pid算法、具体任务要求、ui实现。没有文件夹就自行创建

## 代码规范
1. 只要写入代码就需要添加注释，不要等用户提醒。

2. **注释风格**：统一使用Doxygen标准（`@file`/`@brief`/`@param`/`@return`），只在接口声明和函数内部关键逻辑处标注，保持代码整洁，禁止逐行注释。

3. **变量命名**：简洁易读，不要使用冗长的英文全称，尽量减少三元运算符。在含义明确的上下文中优先使用简短命名。
   - 反例：`interval_ms` → 正例：`ms`
   - 反例：`motor_direction` → 正例：`motor_dir`
   - 原则：能一眼看懂即可，不追求英文完整拼写

## 硬件信息
- 硬件信息存放在hardware_info.md中，需要了解硬件信息的时候请阅读。
- 如果hardware_info.md中的信息与用户输入的信息不符，请返回询问。
